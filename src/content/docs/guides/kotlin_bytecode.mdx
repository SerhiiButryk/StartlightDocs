## Lambda vs anonimus Kotlin classes

Did you know lambda and anonimus classes have different runtime cost?

Let's dive deeper and find out why it happens. 

# Anonimus classes

Let's create a 'Test.kt' file with the source code:

```kotlin
fun interface Listener {
    fun call()
    fun message() = "Hello"
}

class A {
    private var listener: Listener? = null

    fun addListener(listener: Listener) {
        this.listener = listener
    }

    fun call() {
        listener?.message()
    }
}

fun main() {

    val a = A()

    // Creates an anonimus class
    a.addListener(object : Listener {
        override fun call() {
            println("Hello this is my message")
        }
    })

    a.call()

}
```

# Lambda in Kotlin

Now let's create a lambda. This is our new source code:

```kotlin
fun interface Listener {
    fun call()
    fun message() = "Hello"
}

class A {
    private var listener: Listener? = null

    fun addListener(listener: Listener) {
        this.listener = listener
    }

    fun call() {
        listener?.message()
    }
}

fun main() {

    val a = A()

    // Creates a lambda
    val listener = Listener {
        println("Hello this is my message")
    }

    a.addListener(listener)

    a.call()

}
```

Now let's check the bytecode:


Notice 'invokedynamic' instraction which ends with '.end invokedynamic' section. This is the magic!

There is a special static 'LambdaMetafactory.metafactory' method which gets called with some arguments to get a very 

special object which is the 'CallSite' object. Below you can see the 'invokevirtual A call ()V' which is a simple function call.

So, let's sum up what we saw so far:

1. No extra class file 

2. Point of lambda creation is a point of some special handling.

3. The next call (to lambda function) is a simple function call 

Basically the 'LambdaMetafactory.metafactory' method is a one time call. It links a code with a lambda to a special 'CallSite' object

which is able to refenece and coll the code of our lambda.

Wait!! Where is the code of out lamda ? 

Here it is:

```kotlin

        System.out.println((Object) "Hello this is my message");
    }
```

This is the code which will be called in the end.
