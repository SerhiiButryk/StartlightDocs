## Introduction

Debugging Android apps can be difficult if you are not familiar with the code or you can't modify the APK file. This guide explains how to do this more easily.

## Decompile and rebuild APK

1. Download the 'android.py' tool

```bash
$ curl -o android.py https://github.com/SerhiiButryk/DevelopmentTools/blob/main/tools/android.py
```

2. Grant execute permissions to the 'android.py' tool

```bash
$ chmod +x android.py
```

3. Assuming you have placed 'android.py' in the same directory as your APK file, decompile your APK:

```bash
$ android.py -decomp <your file name>.apk
```

Wait while the tool downloads all necessary dependencies. If there are no errors, check the 'output' directory.

4. Modify the sources

5. Rebuild the APK

```bash
$ android.py -build <your folder with modified apk>
```

6. Sign your APK

```bash
$ android.py -resign-apk <your file name>.apk
```

7. Install the APK and enjoy!

```bash
$ adb install <your file name>.apk
```

## Debug existing apk using JADX

**Preconditions:**

1. Add 'android:debuggable="true"' to your application manifest.

```xml
(...)
<application
            android:debuggable="true"
            (...)
            tools:ignore="HardcodedDebugMode">
(...)
</application>
(...)
```

2. Add 'android.os.Debug.waitForDebugger()' right before your code with a breakpoint:

```kotlin
class MainActivity : ComponentActivity() {
    override fun onCreate(savedInstanceState: Bundle?) {
        enableEdgeToEdge()
        super.onCreate(savedInstanceState)

        // ADD THIS
        android.os.Debug.waitForDebugger()

        (...)
    }
    (...)
}
```

OR, if you don't have the sources, you can modify the smali code and rebuild the APK:

```kotlin
invoke-static {}, Landroid/os/Debug;->waitForDebugger()V
```

**Steps to debug:**

1. Launch the emulator and install your APK.

2. Open the installed APK using JADX.

```bash
java -jar jadx.jar <your file name>.apk
```

3. Set a breakpoint in JADX

To do this, find the class, open the 'smali' view, and select 'Show Dalvik bytecode'.

<Image src={s7} alt="" width="600" />

After that, you can set breakpoints directly in the APK bytecode.

<Image src={s8} alt="" width="600" />

4. Launch the app in the emulator.

5. Click on the debug option in JADX, select the process, and double-click on it.

<Image src={s5} alt="" width="400" />

The debugger should now be attached to the running app process.

<Image src={s6} alt="" width="400" />

6. Click the run button.

<Image src={s9} alt="" width="400" />

7. If everything is set up correctly, you'll see the call stack, variable view, and the process will stop at your breakpoint. Now you can debug your app using the JADX debug panel.

<Image src={s10} alt="" width="800" />

**Troubleshoot:**

If the debugger doesn't attach or exits unexpectedly, there may be an issue with adb. Make sure adb is running and listening on port 5037, which is the default port.

If the issue persists, check if port 5037 is not blocked by running:

```bash
lsof -i :5037
```

If there are processes that are also listening to or using port 5037, kill them and restart the JADX debugger.

```bash
kill -9 <PID>
```

You can also kill and restart the adb server:

```bash
adb kill-server
adb start-server
```
